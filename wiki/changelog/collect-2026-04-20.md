## Key Findings

### 1. SpecGuard: Step-Level Verification for Reasoning (arXiv 2604.15244, April 16, 2026)

The most practically significant find for reasoning-heavy deployments. Standard speculative decoding is token-centric — it catches lexical divergences but allows erroneous reasoning steps to propagate. SpecGuard inserts step-level validation using model-internal ensemble verifiers (no external reward model) to catch and recompute bad steps before they corrupt downstream reasoning.

**Key numbers**: +3.6% accuracy, −11% latency vs standard spec decode on reasoning benchmarks; outperforms reward-guided SD as well.

**Practical note**: The verifiers are distilled from the target model's hidden states — this distillation step has a one-time cost and must be redone for new target models. Not yet in vLLM.

### 2. StreamServe: Adaptive Speculation in a Disaggregated Architecture (arXiv 2604.09562, April 2026)

Architecturally interesting because it combines two independently studied optimizations (P/D disaggregation + spec decode) and shows they are complementary rather than competing. The SpecuStream component's insight — that static speculation depth K is suboptimal for reasoning tasks where token entropy fluctuates mid-sequence — is generalizable beyond this specific system.

**Key numbers**: GSM8K 264 tok/s vs 241 tok/s (+9.5%), latency 0.30 s vs 3.50 s (−91%); peak 2235 tok/s on SUM; 11–18× latency reduction overall vs TP-vLLM.

**Caveat**: 320-query evaluation is very small; A800 GPUs only; baseline is vanilla TP-vLLM without native P/D or spec decode.

### 3. Prefill-as-a-Service: Hybrid Attention Reshapes PD Disaggregation Economics (arXiv 2604.15039, April 2026)

Long-term strategic signal. The current assumption that P/D disaggregation requires co-location within a high-bandwidth network domain (NVLink/IB) is predicated on dense-attention models with large KV caches. Hybrid-attention architectures break this assumption by shrinking KV cache volume enough to cross commodity Ethernet. If hybrid architectures become mainstream (which Q2 2026 model releases suggest is underway), the infrastructure model for LLM serving changes significantly.

**Key numbers**: +54% vs homogeneous PD, +32% vs naive heterogeneous, on a 1T-parameter internal hybrid model.

**Limitation**: Single internal model; cross-DC latency (10–100ms RTT) will hurt interactive TTFT use cases.

## Contradictions Found

None. New content is additive:
- SpecGuard's +3.6% accuracy and −11% latency are measured against standard SD in a reasoning context. CSD's 2.33× speedup is measured in a different setup (false-rejection correction for general LLMs). Not directly comparable; complementary claims.
- StreamServe's 11–18× latency reduction has a weaker baseline (TP vLLM, no native P/D, no spec decode) than what vLLM's own P/D stack would show. The relative number is inflated; the SpecuStream adaptive-K insight is still valid independently.
- PrfaaS's throughput improvement is predicated on hybrid-attention model. Dense-attention models (all existing vLLM benchmarks) would not benefit from cross-DC deployment due to KV transfer costs. No contradiction with existing disaggregated-serving content, just a new scenario.
