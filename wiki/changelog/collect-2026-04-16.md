## Key Findings

### TurboQuant KV Cache Compression (PR #38479, merged April 15, 2026)

The most significant inference-optimization change merged to vLLM in this 24-hour window. Key points:

1. **Sub-FP8 KV compression is now in vLLM main** — for the first time, operators can go below 8-bit KV cache without external patches. The `turboquant_k8v4` preset is the conservative entry point.

2. **WHT + Lloyd-Max is the right approach for keys** — Walsh-Hadamard Transform rotation decorrelates outlier channels before scalar quantization, making Lloyd-Max quantization viable at 3–4 bits. This matches the theoretical treatment from the ICLR 2026 Google Research TurboQuant paper.

3. **Asymmetric K/V allocation matters critically** — symmetric 3-bit (equal bits for keys and values) produced 0% GSM8K on community tests. Allocating more bits to keys than values is the right inductive bias. The original PR #38280 missed this; PR #38479 added `tq_k4v3` presets.

4. **Long-context TPOT improvement is real** — at 8K context on Qwen3-4B / RTX PRO 6000 Blackwell, `turboquant_k8v4` improves TPOT from 138.1ms to 135.2ms. This is because higher compression allows more concurrent sequences, improving batching utilization. The benefit outweighs quantization overhead at long contexts.

5. **Not yet in a numbered release** — only on vLLM main. Operators wanting to use this must build from source or wait for v0.20+.

6. **Hybrid model caution** — Qwen3.5-series and other hybrid architectures show more quality degradation. Avoid until per-architecture validation is done.
