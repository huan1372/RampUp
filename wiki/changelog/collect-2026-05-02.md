## Key Findings

### DeepSeek-V4 TileLang Kernel: First Model-Specific Tile Kernel in vLLM (PR #41255)

The `head_compute_mix_kernel` for DeepSeek-V4 MHC attention (PR #41255) is noteworthy beyond its 7–9% throughput improvement for two reasons:

1. **TileLang in production vLLM**: This is the first TileLang-authored kernel merged to vLLM mainline. TileLang is a tile-level DSL analogous to Triton but designed for more explicit tiling-hierarchy control. Its presence in vLLM signals the team's willingness to go beyond CUTLASS and FlashInfer for architecture-specific optimization — consistent with the vLLM IR initiative (the `rms_norm` skeleton in v0.20.0 being the first step of a Helion/IR kernel generation path).

2. **Upstream kernel porting pattern**: The kernel was ported from DeepSeek's own TileKernels repository rather than written from scratch. This is a practical strategy for day-0 model support: let the model creator author the critical kernels, then integrate. Suggests other TileKernels (CSA indexer, HCA compression kernel) may follow the same porting path.

**Benchmark context:** SPEED-Bench (500 prompts, 4×GB200) is a throughput benchmark, not a latency benchmark. The 6% TTFT improvement is the more interesting number for latency-sensitive workloads — it suggests the kernel reduces attention execution time enough to shorten the prefill critical path.

### HMA KV Offload Series Completes After 13 PRs Over ~3 Weeks (PR #41445)

The HMA KV offload series spanned April 22 – May 1, 2026 across 13 PRs, expanding KV offloading from single-group architectures to the full complexity of production models. The series is a case study in incremental enablement:

- Each PR was self-contained and correct at its own level of scope
- The series tackled the problem in natural layers: transfer → store → SWA scheduler → final connector advertisement
- The `SupportsHMA` flag at the end is a clean capability advertisement that downstream tooling (e.g., deployment scripts, model configs) can check

**Production significance:** DeepSeek V4's CSA+HCA+mHC multi-tier KV layout is exactly the class of architecture HMA was designed for. With PR #41445, vLLM can now offload DeepSeek V4 KV cache to CPU — relevant for budget-constrained inference where GPU HBM is smaller than the 1M-context KV footprint.

### MLA Prefill Backend Abstraction: Completes the Backend Abstraction Story (PR #32623)

The default shift from CUTLASS MLA to FlashInfer MLA for prefill is significant because FlashInfer's prefill path is generally faster for variable-length batches (it uses a ragged layout natively, while CUTLASS requires padding). Whether this delivers measurable gains in production depends on prefill batch characteristics — no benchmark comparison was published with the PR.

The cuDNN removal eliminates one more maintenance surface. The old `use_cudnn_prefill` flag was a v0.15-era workaround; its deprecation marks the end of that workaround lifecycle.

**Cross-link to MRV2:** This PR is architecturally consistent with the MRV2 design goal of pluggable backends. With PR #32623, both MLA encode (prefill) and decode paths now support runtime backend selection — completing the backend abstraction design.
