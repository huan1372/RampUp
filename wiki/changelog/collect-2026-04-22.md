## Key Findings

### 1. vLLM Mainline Performance Improvements (April 21–22, 2026)

No new release, but four PRs with cumulative impact:

**PR #40413 — Fused RMS Norm Batch Invariant (April 21):** Removes a dead-code conditional check in `fused_add_rms_norm`. The underlying CUDA kernel already guarantees batch invariance; the conditional was unreachable overhead. Result: **2.1% E2E latency reduction** on Llama-3.1-8B-FP8 (0.524 → 0.513 s). Affects all modern LLMs using fused RMS norm.

**PR #38284 — CUDAGraph Memory Profiling Default (April 21):** Enables CUDA graph memory profiling by default. Primary motivation: DeepSeek-R1 at DP=8, EP configurations triggered startup OOM without profiling. Side effect: **default `--gpu-memory-utilization` is now 0.92** (previously 0.9). This is a user-visible breaking change for deployments that relied on the 0.9 default.

**PR #38877 — MLA + Group FP8 Fusion (April 22):** Fuses per-group FP8 quantization into the MLA attention kernel for DeepSeek V3 and similar MLA-based architectures. B200x4 throughput improvement: 5,345 → 5,367 tok/s (+0.4%). Phase 1 of issue #35792. Also fixes NVFP4 pattern matching for DeepSeek-R1-0528-NVFP4-v2.

**PR #38453 — KV Offload HMA Multi-group (April 22):** Enables the CPU-GPU KV cache offloading handler to work with multi-group KV caches (needed for MLA-based architectures where KV is split into compressed and uncompressed groups). No performance numbers; correctness fix.

### 2. IsoQuant: SO(4) Isoclinic Rotations (arXiv 2603.28430, March 28, 2026)

Late-caught paper (missed in all prior collects). Extends the TurboQuant → RotorQuant rotation lineage to SO(4) isoclinic decomposition. Key idea: any SO(4) rotation = left-isoclinic × right-isoclinic factor; in quaternion form `T(v) = q_L ⊗ v ⊗ q̄_R`. Power-of-2 block size (4D quaternion) aligns with GPU SIMD.

At d=128: IsoQuant-Full achieves **4.49× speedup** over RotorQuant with identical quality; IsoQuant-Fast achieves **4.66×**. FMAs reduced: RotorQuant ~2,408 → IsoQuant-Full 1,024 → IsoQuant-Fast 512 (vs TurboQuant's 16,384 baseline).

No vLLM integration. Research only.

### 3. Sequential KV Cache Compression (arXiv 2604.15356, April 10, 2026)

Deferred from April 21 collect; technical content now confirmed and ingested.

Core claim: per-vector methods (FP8, TurboQuant, IsoQuant) are upper-bounded by per-vector Shannon entropy H(KV_t). The trie-based framework achieves H(KV_t | context), which can be arbitrarily smaller for a good predictor — i.e., the model itself. Two layers: (1) PLT-based semantic prefix deduplication across sessions (generalizes vLLM's prefix caching), (2) predictive delta coding where residuals `KV_t − model_prediction(KV_t)` are stored.

Notable: this theoretical framework makes vLLM's prefix caching a special case of Layer 1. If vLLM adds approximate prefix matching, the framework applies directly.

No benchmark numbers; theoretical paper.
