# Rethinking KV Cache Eviction via a Unified Information-Theoretic Objective

**arXiv:** 2604.25975  
**Authors:** Jiaming Yang, Chenwei Tang, Liangli Zhen, Jiancheng Lv  
**Submitted:** April 28, 2026

## Problem

Standard KV cache eviction methods (H2O, ScissorHands, SnapKV, etc.) use heuristic scoring — typically attention score accumulators — to select which KV entries to drop. These heuristics lack a principled information-theoretic basis, leading to suboptimal retention decisions particularly in long-context tasks where the relationship between attention scores and actual information content diverges.

## Proposed Method: CapKV

The paper rethinks KV eviction through the **Information Bottleneck (IB)** principle. The goal is to find the minimal retained KV subset that maximally preserves information about future attention outputs.

**Formulation:**

Under a linear-Gaussian surrogate of the attention mechanism, the authors derive a closed-form mutual information objective characterizing the effective information capacity of a retained KV cache subset. This yields:

- A theoretically grounded eviction criterion: retain the subset S of KV pairs that maximizes I(Y; KV_S) — the mutual information between the future attention output Y and the retained KV pairs
- Under the linear-Gaussian surrogate, this objective becomes tractable as a log-determinant maximization problem: maximize log det(I + σ^{-2} K_S^T K_S) over subset S, where K_S is the key matrix of retained entries

**CapKV Implementation:** Approximates the log-determinant objective using **statistical leverage scores** — diagonal elements of the hat matrix H = K(K^T K)^{-1} K^T. Leverage scores identify which KV entries are most "influential" in the attention computation (high leverage = hard to reconstruct from remaining entries). This is an O(n) approximation to the NP-hard exact subset selection.

## Key Properties

- **Unified objective**: single IB-derived criterion handles both the "importance to current query" and "information capacity" aspects that prior methods address with separate ad-hoc rules
- **Training-free**: no model modifications or calibration datasets required
- **Theoretically grounded**: IB formulation provides a principled basis for comparing eviction methods
- **Comparison to prior art**: attention-score accumulators (H2O, SnapKV) approximate a different quantity — a proxy for historical query relevance, not future information capacity. These proxies diverge at long contexts.

## Results

Extensive experiments across multiple models (exact model names not captured in search results) on long-context benchmarks:

- **CapKV consistently outperforms prior eviction methods** (H2O, SnapKV, ScissorHands) across all tested compression ratios and models
- Better memory efficiency vs accuracy tradeoff than heuristic baselines
- Performance gains largest at high compression ratios (aggressive eviction) where heuristic scoring breaks down

## vLLM Integration

No vLLM integration. Eviction-based KV management requires modifications to the KV cache allocator (PagedAttention block manager) to implement the leverage score computation and block selection at each step. CapKV's O(n) approximation is cheap enough to run per-step, but the integration engineering is non-trivial.

## Relationship to Existing Work in KB

- **GRACE (arXiv 2604.16983)**: graph-guided channel elimination — removes entire KV channels (structural pruning); CapKV retains all channels but drops entire token-level entries (temporal eviction). Different axes of compression.
- **TurboQuant/KV quantization**: quantizes retained KV values; CapKV decides which entries to retain. Orthogonal; can be composed.
- **TTKV (arXiv 2604.19769)**: temporal tiering (recent=HBM, old=DRAM); no eviction — all tokens retained. Different approach.
- **Sequential KV compression (arXiv 2604.15356)**: delta coding / trie compression of retained entries. Orthogonal to eviction.
