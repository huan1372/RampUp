# vLLM v0.19.1 Release Notes

**Version**: v0.19.1  
**Released**: April 18, 2026  
**Type**: Patch release on top of v0.19.0 (April 3, 2026)

## Summary

v0.19.1 is a patch release focused on Transformers v5 compatibility and Gemma4 bug fixes. The critical motivation: v0.19.0 as shipped on PyPI pinned `transformers<5`, but Gemma4 support requires `transformers>=5.5.0`. v0.19.1 resolves this by upgrading to Transformers v5.5.3 (some sources say v5.5.4).

## Key Changes

### Dependency Upgrade
- Upgraded `transformers` pin from `>=4.56.0,<5` to `>=5.5.3` (PR #30566 referenced for the v5 transition)
- This unblocks Gemma4 users who install via PyPI without manual transformers overrides

### New Model Features
- **Gemma4 Eagle3 support** (PR #39450) — EAGLE-3 speculative decoding drafter for Gemma4 models
- **Quantized MoE for Gemma4** (PR #39045) — FP8/quantized mixture-of-experts inference for Gemma4 architecture

### Bug Fixes (Gemma4 streaming and tool calls)
- Fix invalid JSON in Gemma4 streaming tool calls (removed partial delimiters) (PR #38992)
- Fix Gemma4 streaming HTML duplication after tool calls (PR #38909)
- Fix Gemma4 streaming tool call corruption for split boolean/number values (PR #39114)
- Enable Gemma4ForCausalLM to load LoRA adapters (PR #38844)
- Fix Gemma4 tool parser converting bare `null` to string `"null"` (PR #39679)
- Fix Gemma4 token repetition via dynamic BOS injection for PT models (PR #39842)

### Other Fixes
- Resolved Kimi K25 media placeholder token ID resolution (PR #39344)
- Reasoning parser updates

## Context: v0.19.0 (April 3, 2026)

v0.19.0 was the major release that v0.19.1 patches. Key v0.19.0 features (for context):
- **Zero-bubble async scheduling with speculative decoding** — PR #32951; async scheduling now overlaps draft generation with target model forward passes
- **MRV2 default** — Model Runner V2 became the default execution core (previously opt-in since v0.18.0)
- **Gemma4 architecture support** (all variants: MoE, multimodal, reasoning, tool use)
- **General CPU KV cache offloading** with pluggable cache policy interface (PRs #37160, #37874, #34805, #36642, #37853)

## vLLM April 19-21, 2026 PRs (background maintenance)

Several merged PRs in the April 19-21 window are primarily maintenance/bugfixes, none individually high enough signal for standalone ingestion:
- PR #40167: IR op testing/benchmarking infrastructure
- PR #40152: mxfp8 online quant migrated to new frontend
- PR #40331: Parallelize torch/transformers import + weight prefetch + forkserver prewarm (startup acceleration — subsequently reverted in PR #40438)
- PR #40411: Gemma4 multimodal embedder norm order fix

## Relevance to Knowledge Base

- v0.19.1 closes the Transformers v5 pinning issue — operators can now install from PyPI and get Gemma4 support without manual dependency overrides
- Gemma4 Eagle3 (PR #39450) expands the Eagle drafter coverage to a new model family; this is the first Eagle-family drafter for a Google-origin MoE+multimodal architecture
- Quantized MoE for Gemma4 demonstrates that FP8/quantization is now expected to work across MoE-based inference, not just dense models
